# WORKLOG convention

Maintain a WORKLOG.md in project roots to track progress and next steps.

Format: task plan sections at the top, with a dated log (diary-style entries per session) appended at the bottom. Check off task items as they're completed, and clean up completed sections in bulk once they've been logged.

Don't use checkboxes alone to track progress — add a log entry explaining what was done and any notable findings. Prefer narrative over bare checklists.
